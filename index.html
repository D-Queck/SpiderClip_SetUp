What is SpiderClip?

Smartwatches and fitness trackers integrate different sensors from inertial measurement units to heart rate sensors in a very compact and affordable form factor. This makes them interesting and relevant research tools. One potential application domain is virtual reality, e.g., for health related applications such as exergames or simulation approaches. However, commercial devices complicate and limit the collection of raw and real-time data, suffer from privacy issues and are not tailored to using them with VR tracking systems. We address these issues with an open source design to facilitate the construction of VR-enabled wearables for conducting scientific experiments.

System

The system consists of three subsystems: a Hardware Sensor Platform, a Communication Interface, and a VR-Wearable.

Hardware Sensor Platform

Captures biometric and motion data through integrated sensors, designed for real-time streaming.

Communication Interface

Provides wireless BLE communication to relay data efficiently between hardware and host system.

VR-Wearable

An interactive VR model for data visualization and simulation within immersive training setups.

System Design

The following schematic model illustrates how the SpiderClip system is structured and how its subsystems interact.

Simplified schematic representation of the simulation system. It is integrated into a VR system with tracking capabilities. On the left: the hardware sensor platform (4.5.1) for capturing and processing sensor data. Also, the tracking system, which uses base stations to track the position of the wearable and sends the data to the PC via dongle. In the center: the communication interface (4.5.2), which acts as the master of the BLE connection and serves as a serial bridge. It enables wireless communication between the hardware sensor platform and the VR wearable. On the right: the VR wearable (4.5.3), which is integrated at the application layer into a Unity project.

Hardware-Sensor-Platform

The hardware sensor platform acts as the periphery of the communication interface. It consists of a 3D-printed housing and a sensor platform with a communication module.
